# 贡献指南

本 Wiki 的所有内容都是普通的 **Markdown 文件**，存放在 `wiki/public/docs/` 目录下。
**你只管写 Markdown，导航、首页卡片、搜索都会自动更新** —— 不需要改任何前端代码。

## 三步新增一篇文档

1. 在对应分类文件夹里新建一个 `.md` 文件，例如 `生活篇/如何办理银行卡.md`。
2. 写正文。第一行的 `# 标题` 会自动成为页面标题。
3. 提交 / 保存即可。本地开发时（`npm run dev`）保存后导航和搜索会**热更新**，无需重启。

> 想新建一个分类？直接新建一个文件夹再往里放 `.md` 文件就行。

## 可选：用 Frontmatter 精细控制

在文件最顶部加一段 YAML，可以覆盖默认行为（全部字段都是可选的）：

```yaml
---
title: 行前指南          # 不写则取正文第一个 # 标题
icon: ✈️                # 侧边栏 / 卡片上的图标
order: 1                # 同级排序，数字越小越靠前
description: 出发前的准备  # 不写则自动截取首段
tags: [新生, 准备]        # 用于站内搜索
draft: false            # 设为 true 会标记为「草稿」
---
```

## 可选：为分类命名 / 排序

在分类文件夹里放一个 `_category.json`：

```json
{
  "label": "入学篇",
  "icon": "✈️",
  "order": 1,
  "description": "行前准备、签证、机场指南……"
}
```

## 图片怎么放

把图片放进该分类下的 `img/` 文件夹，再用**相对路径**引用：

```markdown
![校园地图](img/Campus_Map.jpg)
```

系统会自动把它解析成正确的访问地址，亮色 / 暗色模式都能正常显示。

## 背后发生了什么（了解即可）

- 构建脚本 `scripts/generate-manifest.mjs` 会扫描 `public/docs/`，把每篇文档的标题、图标、排序、标签、小标题、最后更新时间汇总进 `src/wiki.data.js`。
- `npm run dev` / `npm run build` 会自动运行它；开发时还有一个 Vite 插件**监听文件变化并实时重建**。
- 前端的侧边栏、首页、搜索、面包屑、上一篇/下一篇全部读取这份清单 —— 所以你加一篇文档，处处自动生效。
